Glossary

A quick mental model for how the Orchestrator is structured.

Workspace
A saved configuration that groups worktrees/terminals for a project (or many repos in a mixed workspace). Opening a workspace starts its sessions.
Worktree
A git worktree folder (e.g. work6) on disk. One worktree usually has an Agent terminal and (optionally) a paired Server terminal.
Terminal / Session
A running process (PTY) owned by the Orchestrator. It can be recovered after restart unless you explicitly close it.
Agent vs Server
Agent runs Claude/Codex work. Server runs your dev server (game/app). They’re paired per worktree and should generally live/die together.
Close vs Remove
Close terminal (×): stops the running processes for that worktree (agent + server). The worktree stays in the workspace and can be re-opened.
Remove worktree (🗑): removes the worktree from the workspace config, closes its terminals, and clears recoverables. Files on disk are kept.
Close chat: marks thread status as closed (and can optionally close linked sessions via API flag).
Archive chat: marks thread status as archived (and can optionally close linked sessions via API flag).
