What it is

A workspace layer for many local agent sessions.

This project is less about a prettier terminal and more about coordination: repositories, worktrees, session state, review loops, and recovery all in one place.

Control surface

Create workspaces, mix repositories, open browser-like tabs, and keep each terminal pair mapped to a worktree.

Operational awareness

Track busy, waiting, and ready states, surface PR links, and route work through Commander, queue, and review tools.

Local by default

The stack runs on your machine with loopback-oriented defaults, local workspace data, and a native Tauri wrapper for desktop use.

How it works

From repositories to a reviewable session map.

01

Choose repositories

Scan your local GitHub tree or point the workspace wizard at a repo directly.

02

Split into worktrees

Map terminal pairs to worktrees so multiple branches and tasks stay isolated.

03

Run and monitor agents

Launch Claude Code or Codex, keep tabs alive, and watch state changes in real time.

04

Review and ship

Open PRs, use the diff viewer, queue reviews, and coordinate sessions from Commander.

Architecture

Three layers, one operating surface.

Server

Express + Socket.IO

Session lifecycle, workspace state, git/worktree operations, review APIs, recommendations, and recovery services.

Client

Vanilla JS terminal UI

Terminal grids, tabbed workspaces, wizards, board views, Commander panel, and live status rendering in the browser.

Desktop

Tauri app shell

Native packaging, backend bootstrapping, local auth handoff, and a desktop-friendly experience when a browser is not enough.

How to use it

Get the web build running, then create your first workspace.

01

Install the app

Clone the repo, create your `.env`, and install root plus diff-viewer dependencies.

02

Start the local stack

Run the web mode for browser use, or build the Tauri app if you want a desktop shell.

03

Create a workspace

Use the dashboard wizard, add worktrees, and start your agent/server session pairs.

Quick start Web mode 
git clone https://github.com/web3dev1337/claude-orchestrator.git
cd claude-orchestrator

cp .env.example .env
npm install
cd diff-viewer && npm install && cd ..

npm run dev

Then open the local client, launch the workspace wizard, and choose how many worktree-backed terminal pairs you want.

Why this is interesting

It treats agent work as an operating environment, not a single chat box.

Local control

Designed for machine-local orchestration, with loopback-safe defaults and direct terminal ownership.

Scale without clutter

Workspaces, worktrees, and tabs keep many parallel tasks visible without collapsing into window sprawl.

Review stays inside the tool

PR links, queueing, activity, and the diff viewer sit next to the sessions that produced the code.

Not locked to one agent

Claude Code and Codex fit into the same operational model, with room for more providers over time.

Showcase source is isolated in /site so the eventual GitHub Pages surface can stay separate from the repo’s internal documentation tree.
